# Документация по Vision Assistant Pro

**Vision Assistant Pro** — это продвинутый мультимодальный ИИ-ассистент для NVDA. Он использует первоклассные ИИ-движки для интеллектуального чтения с экрана, перевода, голосового ввода и анализа документов.

_Это дополнение было выпущено для сообщества в честь Международного дня инвалидов._

## 1. Установка и настройка

Перейдите в **Меню NVDA > Параметры > Настройки > Vision Assistant Pro**.

### 1.1 Настройки подключения
- **Провайдер:** Выберите предпочитаемый ИИ-сервис. Поддерживаемые провайдеры: **Google Gemini**, **OpenAI**, **Mistral**, **Groq**, **MiniMax** и **Custom** (серверы, совместимые с OpenAI, например, Ollama, LM Studio, Jan.ai или KoboldCPP).
- **Важное примечание:** Мы настоятельно рекомендуем использовать **Google Gemini** для наилучшей производительности и точности (особенно для анализа изображений/файлов).
- **API-ключ:** Обязательное поле. Вы можете ввести несколько ключей (разделяя их запятыми или переводами строк) для автоматической ротации.
- **Получить модели:** После ввода API-ключа нажмите эту кнопку, чтобы загрузить последний список доступных моделей от провайдера.
- **Модель ИИ:** Выберите основную модель для общего чата и анализа.

### 1.2 Расширенная маршрутизация моделей
*Доступно для всех провайдеров, включая Gemini, OpenAI, Groq, Mistral и Custom.*

> **⚠️ Предупреждение:** Эти настройки предназначены **только для опытных пользователей**. Если вы не уверены, для чего предназначена конкретная модель, оставьте этот параметр **отключенным**. Выбор несовместимой модели для задачи (например, текстовой модели для Vision) вызовет ошибки и остановит работу дополнения.

Установите флажок **"Расширенная маршрутизация моделей (для конкретных задач)"**, чтобы получить детальный контроль. Это позволит вам выбирать конкретные модели из раскрывающегося списка для разных задач:
- **OCR / Модель Vision:** Выберите специализированную модель для анализа изображений.
- **Распознавание речи (STT):** Выберите конкретную модель для диктовки.
- **Синтез речи (TTS):** Выберите модель для генерации аудио.
- **Модель AI Operator:** Выберите конкретную модель для задач автономного управления компьютером.
*Примечание: Неподдерживаемые функции (например, TTS для Groq) будут автоматически скрыты.*

### 1.3 Расширенная настройка конечных точек (Пользовательский провайдер)
*Доступно только при выборе "Custom".*

> **⚠️ Предупреждение:** Этот раздел позволяет вручную настроить API и предназначен для **продвинутых пользователей**, запускающих локальные серверы или прокси. Неправильные URL-адреса или имена моделей нарушат подключение. Если вы точно не знаете, что это за конечные точки, оставьте этот параметр **отключенным**.

Установите флажок **"Расширенная настройка конечных точек"** для ручного ввода данных сервера. В отличие от встроенных провайдеров, здесь необходимо **вводить** конкретные URL-адреса и имена моделей:
- **URL списка моделей:** Конечная точка для получения доступных моделей.
- **URL конечной точки OCR/STT/TTS:** Полные URL-адреса для конкретных сервисов (например, `http://localhost:11434/v1/audio/speech`).
- **Пользовательские модели:** Вручную введите имя модели (например, `llama3:8b`) для каждой задачи.

### 1.3.1 Настройка локального ИИ (Настройка в одно действие)
Чтобы сделать интеграцию локального, полностью автономного ИИ максимально простой, в настройках пользовательского провайдера доступна специальная кнопка **"Настройка локального ИИ"**.

Если вы запускаете сервер локальной ИИ-модели на своем компьютере:
1. Выберите **Custom** в качестве провайдера.
2. Нажмите кнопку **"Настройка локального ИИ"**.
3. Выберите ваш локальный ИИ-движок из доступного диалога:
   - **Ollama** (по умолчанию `http://127.0.0.1:11434`)
   - **LM Studio** (по умолчанию `http://127.0.0.1:1234`)
   - **Jan.ai** (по умолчанию `http://127.0.0.1:1337`)
   - **KoboldCPP** (по умолчанию `http://127.0.0.1:5001`)
4. Дополнение мгновенно настроит правильный локальный URL, тип API и автоматически получит ваши активные автономные модели для заполнения поля выбора **"Модель ИИ"**.

*Примечание о сети и прокси:* Этот механизм локального подключения имеет расширенный механизм обхода прокси. Даже если у вас активен системный VPN или прокси в режиме TUN, ваши локальные запросы к ИИ будут полностью обходить его, обеспечивая стабильные автономные подключения без ошибок 502 Bad Gateway.

### 1.4 Общие настройки
- **OCR-движок:** Выберите между **Chrome (Быстрый)** для получения быстрых результатов или **ИИ (Расширенный)** для лучшего сохранения структуры.
    - *Примечание:* Если вы выберете "ИИ (Расширенный)", но ваш провайдер установлен на OpenAI/Groq, дополнение интеллектуально перенаправит изображение на модель зрения вашего активного провайдера.
- **Голос TTS:** Выберите предпочитаемый стиль голоса. Этот список динамически обновляется в зависимости от вашего активного провайдера.
- **Креативность (Температура):** Управляет случайностью ответов ИИ. Более низкие значения лучше подходят для точного перевода/OCR.
- **URL прокси:** Настройте, если ИИ-сервисы ограничены в вашем регионе (поддерживаются локальные прокси, например `127.0.0.1` или мостовые URL).

### 2.1 Горячие клавиши средства чтения документов (В окне просмотра)
- **Ctrl + PageDown:** Перейти на следующую страницу.
- **Ctrl + PageUp:** Перейти на предыдущую страницу.
- **Alt + A:** Открыть диалоговое окно чата, чтобы задать вопросы о документе.
- **Alt + R:** Принудительно **повторно сканировать с помощью ИИ**, используя вашего активного провайдера.
- **Alt + G:** Сгенерировать и сохранить высококачественный аудиофайл (WAV/MP3). *Скрыто, если провайдер не поддерживает TTS.*
- **Alt + S / Ctrl + S:** Сохранить извлеченный текст в виде файла TXT или HTML.

## 3. AI Operator — Автономное управление компьютером

**AI Operator** превращает Vision Assistant Pro из пассивного средства чтения в активного помощника, который может взаимодействовать с вашим компьютером от вашего имени. Вы можете попросить его описать экран, ответить на вопросы о том, что он видит, или даже взять управление на себя — нажимать кнопки, перетаскивать элементы, вводить текст и перемещаться по приложениям с помощью команд на естественном языке.

Самое большое преимущество? Он отлично работает в совершенно недоступном программном обеспечении. Если вы застряли в пользовательском приложении, на удаленном рабочем столе или на веб-сайте, где ваша программа чтения с экрана полностью молчит, оператору это не мешает. Поскольку он "видит" экран визуально, он может находить, читать и взаимодействовать с элементами, у которых нет нулевых меток доступности.

### Как это работает
1. Нажмите **NVDA + Shift + V**, затем нажмите **Shift + A** (или используйте прямое сочетание клавиш), чтобы открыть диалог AI Operator.
2. Введите на простом языке, что вы хотите сделать (например, "Нажми кнопку Сохранить", "Что говорит сообщение об ошибке?" или "Переименуй файл в final.pdf").
3. ИИ проанализирует ваш экран, определит соответствующие элементы и выполнит действие или предоставит ответ. Если задача требует нескольких шагов, оператор будет продолжать работу до ее завершения.
4. Нажмите **Shift + A** еще раз в любой момент, чтобы мгновенно прервать выполняемую операцию.

### Поддерживаемые действия
Оператор понимает широкий спектр команд:
- **Описать и ответить**: "Опиши макет экрана" или "Что говорит сообщение об ошибке?"
- **Нажать**: "Нажми кнопку Сохранить"
- **Нажать правой кнопкой**: "Нажми правой кнопкой на файл"
- **Дважды нажать**: "Дважды нажми на документ"
- **Перетащить**: "Перетащи документ в папку Архив"
- **Ввести**: "Введи 'Привет мир' в поле поиска"
- **Прокрутить**: "Прокрути вниз три раза"
- **Нажать клавишу**: "Нажми Enter", "Нажми Tab", "Нажми Escape"
- **Многошаговые задачи**: "Открой Проводник, найди отчет и переименуй его в final.pdf"

### Важные примечания
- **⚠️ Предупреждение об использовании API**: Поскольку оператору нужно точно "видеть", что происходит на экране, он отправляет скриншот высокого разрешения с каждым шагом. Частое использование будет потреблять вашу квоту API значительно быстрее, чем стандартные текстовые функции.
- **Приложения с правами администратора**: Если NVDA запущена без прав администратора, оператор может не иметь возможности взаимодействовать с окнами, требующими повышенных привилегий. Это ограничение безопасности Windows, а не ошибка дополнения.
- **Рекомендации**: Для достижения наилучших результатов давайте четкие и конкретные команды. "Нажми синюю кнопку Отправить внизу формы" почти всегда будет работать лучше, чем просто "Нажми кнопку".

## 4. Пользовательские запросы и переменные

Вы можете управлять запросами в **Настройки > Запросы > Управление запросами...**.

### Поддерживаемые переменные
- `[selection]`: Выделенный в данный момент текст.
- `[clipboard]`: Содержимое буфера обмена.
- `[screen_obj]`: Скриншот объекта навигации.
- `[screen_fg_obj]`: Скриншот активного окна на переднем плане.
- `[screen_full]`: Полноэкранный скриншот.
- `[file_ocr]`: Выбрать файл изображения/PDF для извлечения текста.
- `[file_read]`: Выбрать документ для чтения (TXT, Код, PDF).
- `[file_audio]`: Выбрать аудиофайл для анализа (MP3, WAV, OGG).

***
**Примечание:** Для всех функций ИИ требуется активное подключение к Интернету. Многостраничные документы обрабатываются автоматически.

## Изменения в версии 7.0.0

*   **Возобновление прерванных сканирований**: Добавлена функция возобновления как для Чтения документов, так и для Умных действий с файлами. Если сканирование было прервано, теперь вы можете продолжить с того места, где остановились, вместо того чтобы начинать заново.
*   **Новая переменная `[screen_fg_obj]`**: Добавлена пользовательская переменная запроса для захвата скриншота только активного окна на переднем плане, а не всего экрана.
*   **Умные повторы и ротация ключей**: Дополнение теперь автоматически повторяет попытки до 5 раз на одном ключе при возникновении временных перегрузок сервера (например, "высокий спрос" или ошибочные ответы). Если повторные попытки не удаются, оно автоматически переключается на следующий API-ключ в вашем списке.
*   **Обнаружение шторки экрана**: Добавлена проверка для предотвращения создания скриншотов, когда активна Шторка экрана (независимо от того, включена ли она постоянно или временно активирована горячей клавишей). Дополнение предупредит вас и остановится, предотвращая отправку черных изображений и бесполезную трату токенов API.
*   **Изменения в Чтении документов**: Диалог диапазона PDF теперь автоматически предварительно выбирает целевой язык из ваших настроек дополнения. Также улучшена обработка потоков, чтобы фоновые задачи корректно останавливались при закрытии средства чтения.
*   **Интеграция нативного OCR Mistral**: Интегрирован собственный OCR API Mistral для документов. Многостраничные документы автоматически объединяются, загружаются и обрабатываются пакетами с использованием специализированной конечной точки `/v1/ocr` Mistral, в то время как одностраничные изображения обрабатываются напрямую без ненужных преобразований в PDF.
*   **Динамические обработчики пользовательских URL**: Изменение пользовательского URL API теперь мгновенно очищает кэш списка моделей и восстанавливает поле ручного ввода модели. Это обеспечивает полную совместимость с пользовательскими конечными точками (такими как Cloudflare AI Gateway), которые не поддерживают стандартную конечную точку `/v1/models` для списка.
*   **Переработанный движок ввода AI Operator**: Полностью переписанная базовая система эмуляции мыши и клавиатуры для AI Operator. Устаревший API `mouse_event` заменен на современный Windows API `SendInput`, что обеспечивает значительно более высокую совместимость с современными приложениями, окнами под защитой UAC и дисплеями с высоким DPI.
*   **Исправлены операции перетаскивания**: Операции перетаскивания в AI Operator теперь полностью стабильны и надежны. Новый движок использует естественные кривые "сглаживания", точное позиционирование курсора, оптимизированную синхронизацию и интеллектуальную технику "подталкивания", чтобы гарантировать, что Windows и приложения правильно распознают и выполняют жесты перетаскивания без сбоев на полпути.
*   **Поддержка нескольких мониторов**: AI Operator теперь полностью поддерживает конфигурации с несколькими мониторами. Движения мыши и клики корректно работают на всех мониторах с использованием флага `MOUSEEVENTF_VIRTUALDESK`, обеспечивая точное позиционирование независимо от того, на каком мониторе находится целевое приложение.
*   **Улучшенная эмуляция клавиатуры**: Улучшена эмуляция нажатий клавиш для полной поддержки "Расширенных клавиш" (таких как клавиши со стрелками, Home, End, Page Up/Down, Insert, Delete и F1-F12). Это гарантирует, что команды навигации и сочетания клавиш, отправляемые AI Operator, работают безупречно во всех приложениях.
