# Документация по Vision Assistant Pro

**Vision Assistant Pro** — это продвинутый многофункциональный ИИ-ассистент для NVDA. Он использует лучшие в мире ИИ-движки для интеллектуального чтения с экрана, перевода, голосового ввода и анализа документов.

_Это дополнение было выпущено для сообщества в честь Международного дня инвалидов._

## 1. Установка и настройка

Откройте **Меню NVDA > Параметры > Настройки > Vision Assistant Pro**.

### 1.1 Параметры подключения
- **Провайдер:** Выберите предпочитаемый ИИ-сервис. Поддерживаемые провайдеры: **Google Gemini**, **OpenAI**, **Mistral**, **Groq**, **MiniMax** и **Пользовательский** (серверы, совместимые с OpenAI, например Ollama, LM Studio, Jan.ai или KoboldCPP).
- **Важное примечание:** Мы настоятельно рекомендуем использовать **Google Gemini** для наилучшей производительности и точности (особенно для анализа изображений и файлов).
- **API-ключ:** Требуется. Вы можете ввести несколько ключей (разделённых запятыми или новыми строками) для автоматической смены.
- **Получить модели:** После ввода API-ключа нажмите эту кнопку, чтобы загрузить список доступных моделей от провайдера.
- **ИИ-модель:** Выберите основную модель для общего чата и анализа.

### 1.2 Продвинутая маршрутизация моделей
*Доступно для всех провайдеров, включая Gemini, OpenAI, Groq, Mistral и Пользовательского.*

> **⚠️ Предупреждение:** Эти настройки предназначены **только для продвинутых пользователей**. Если вы не уверены, что делает конкретная модель, оставьте эту опцию **отключённой**. Выбор несовместимой модели для задачи (например, текстовой модели для зрения) приведёт к ошибкам и остановит работу дополнения.

Отметьте **"Продвинутая маршрутизация моделей (для конкретных задач)"**, чтобы получить детальный контроль. Это позволит вам выбирать конкретные модели из раскрывающегося списка для разных задач:
- **Модель для OCR / зрения:** Выберите специализированную модель для анализа изображений.
- **Распознавание речи (STT):** Выберите модель для диктовки.
- **Синтез речи (TTS):** Выберите модель для генерации аудио.
- **Модель ИИ-оператора:** Выберите модель для автономного выполнения задач на компьютере.
*Примечание: Неподдерживаемые функции (например, синтез речи для Groq) будут автоматически скрыты.*

### 1.3 Расширенная настройка адресов (пользовательский провайдер)
*Доступно только при выборе "Пользовательский".*

> **⚠️ Предупреждение:** Этот раздел позволяет вручную настраивать API и предназначен для **опытных пользователей**, запускающих локальные серверы или прокси. Неправильные адреса или имена моделей нарушат подключение. Если вы точно не знаете, что это за адреса, оставьте эту опцию **отключённой**.

Отметьте **"Расширенная настройка адресов"**, чтобы вручную ввести данные сервера. В отличие от стандартных провайдеров, здесь необходимо **вводить** конкретные адреса и имена моделей:
- **Адрес списка моделей:** Адрес для получения списка доступных моделей.
- **Адреса для OCR/STT/TTS:** Полные адреса для конкретных сервисов (например, `http://localhost:11434/v1/audio/speech`).
- **Пользовательские модели:** Вручную введите имя модели (например, `llama3:8b`) для каждой задачи.

### 1.3.1 Настройка локального ИИ (настройка в одно действие)
Чтобы упростить интеграцию полностью локального офлайн-ИИ, в настройках пользовательского провайдера доступна специальная кнопка **"Настроить локальный ИИ"**.

Если вы запускаете локальный ИИ-сервер на вашем компьютере:
1. Выберите **Пользовательский** в качестве провайдера.
2. Нажмите кнопку **Настроить локальный ИИ**.
3. Выберите ваш локальный ИИ-движок в появившемся диалоге:
   - **Ollama** (по умолчанию `http://127.0.0.1:11434`)
   - **LM Studio** (по умолчанию `http://127.0.0.1:1234`)
   - **Jan.ai** (по умолчанию `http://127.0.0.1:1337`)
   - **KoboldCPP** (по умолчанию `http://127.0.0.1:5001`)
4. Дополнение мгновенно настроит правильный локальный адрес, тип API и автоматически загрузит ваши активные офлайн-модели, чтобы заполнить поле выбора **ИИ-модель**.

*Примечание о сети и прокси:* Этот движок локального подключения имеет встроенный механизм обхода прокси. Даже если у вас активен системный VPN или TUN-прокси, запросы к локальному ИИ будут полностью его обходить, обеспечивая стабильные офлайн-подключения без ошибок "502 Bad Gateway".

### 1.4 Общие настройки
- **Движок OCR:** Выберите между **Chrome (быстрый)** для быстрых результатов и **ИИ (продвинутый)** для лучшего сохранения форматирования.
    - *Примечание:* Если вы выберете "ИИ (продвинутый)", но ваш провайдер установлен на OpenAI/Groq, дополнение интеллектуально перенаправит изображение на визион-модель вашего активного провайдера.
- **Голос TTS:** Выберите предпочтительный голос. Этот список обновляется динамически в зависимости от активного провайдера.
- **Креативность (Температура):** Управляет случайностью ИИ. Более низкие значения лучше подходят для точного перевода или OCR.
- **Адрес прокси:** Настройте это, если ИИ-сервисы ограничены в вашем регионе (поддерживаются локальные прокси, например `127.0.0.1`, или мостовые адреса).

## 2. Уровень команд и горячие клавиши

Чтобы избежать конфликтов клавиш, это дополнение использует **уровень команд**.
1. Нажмите **NVDA + Shift + V** (главная клавиша), чтобы активировать уровень (вы услышите звуковой сигнал).
2. Отпустите клавиши, затем нажмите одну из следующих клавиш:

| Клавиша     | Функция                     | Описание                                                                                                 |
|-------------|-----------------------------|----------------------------------------------------------------------------------------------------------|
| **Shift + A** | **ИИ-оператор**            | **Автономное управление:** Скажите ИИ, что сделать на экране. Повторное нажатие немедленно прерывает операцию. |
| **E**         | **Обозреватель интерфейса**            | **Интерактивный клик:** Определяет и нажимает элементы интерфейса в любом приложении.                         |
| **T**         | Умный переводчик           | Переводит текст под курсором навигатора или выделенный текст.                                            |
| **Shift + T** | Перевод буфера обмена       | Переводит содержимое буфера обмена.                                                                      |
| **R**         | Уточнение текста               | Суммирует, исправляет грамматику, объясняет или выполняет **пользовательские запросы**.      |
| **V**         | Зрение объекта              | Описывает текущий объект навигатора.                                                                     |
| **O**         | Зрение всего экрана         | Анализирует весь экран: макет и содержимое.                                                              |
| **Shift + V** | Анализ онлайн-видео      | Анализирует видео с **YouTube**, **Instagram**, **TikTok** или **Twitter (X)**.                               |
| **D**         | Чтение документов            | Расширенное чтение PDF и изображений с выбором диапазона страниц.                                        |
| **F**         | **Умное действие с файлом**      | Контекстно-зависимое распознавание выбранного файла изображения, PDF или TIFF.                               |
| **A**         | Расшифровка аудио        | Преобразует файлы MP3, WAV или OGG в текст.                                                              |
| **C**         | Решение CAPTCHA             | Захватывает и решает CAPTCHA (поддерживает порталы госуслуг).                                            |
| **S**         | Умная диктовка                    | Преобразует речь в текст. Нажмите, чтобы начать запись, нажмите снова, чтобы остановить и ввести текст.     |
| **Control+L** | **Живой помощник**         | **Живой помощник (только Gemini):** Запускает или завершает живой голосовой и экранный разговор с ИИ-ассистентом. |
| **I**         | Состояние           | Сообщает текущий прогресс (например, "Сканирование...", "Ожидание").                                   |
| **L**         | **Разметить объект**           | **Семантическая ИИ-разметка:** Постоянно помечает текущий элемент / иконку.                                 |
| **Shift + L** | **Управление / сканирование меток**     | Открывает менеджер меток (если метки существуют) или сканирует приложение на наличие безымянных элементов.   |
| **U**         | Проверка обновлений               | Вручную проверяет на GitHub наличие новой версии дополнения.                                             |
| **Пробел**     | Повтор последнего результата         | Показывает последний ответ ИИ в диалоговом окне для просмотра или уточняющих вопросов.                   |
| **H**         | Справка по командам              | Отображает список всех доступных сочетаний клавиш.                                                       |

### 2.1 Горячие клавиши чтения документов (внутри окна просмотра)
- **Ctrl + PageDown:** Перейти на следующую страницу.
- **Ctrl + PageUp:** Перейти на предыдущую страницу.
- **Alt + A:** Открыть чат, чтобы задать вопросы о документе.
- **Alt + R:** Принудительно **пересканировать с помощью ИИ**, используя вашего активного провайдера.
- **Alt + G:** Сгенерировать и сохранить высококачественный аудиофайл (WAV/MP3). *Скрыто, если провайдер не поддерживает синтез речи.*
- **Alt + S / Ctrl + S:** Сохранить извлечённый текст в файл TXT или HTML.

## 3. Пользовательские запросы и переменные

Вы можете управлять запросами в **Параметры > Запросы > Управление запросами...**.

### Поддерживаемые переменные
- `[selection]`: Выделенный текст.
- `[clipboard]`: Содержимое буфера обмена.
- `[screen_obj]`: Снимок экрана объекта навигатора.
- `[screen_full]`: Снимок всего экрана.
- `[file_ocr]`: Выбрать файл изображения или PDF для извлечения текста.
- `[file_read]`: Выбрать документ для чтения (TXT, код, PDF).
- `[file_audio]`: Выбрать аудиофайл для анализа (MP3, WAV, OGG).

***
**Примечание:** Для всех функций ИИ требуется активное подключение к Интернету. Многостраничные документы обрабатываются автоматически.

## Изменения в версии 6.5.0

*   **Живой помощник:** Добавлен помощник в реальном времени с голосом и экраном, доступный исключительно для провайдера Google Gemini (или совместимых с Gemini пользовательских провайдеров). Включает интерактивную голосовую настройку и настройку глубины мышления прямо внутри диалога, с автоматическим переподключением при изменении настроек.
*   **Провайдер MiniMax AI:** Интегрирован MiniMax как полноценный провайдер с полной мультимодальной поддержкой (чат, зрение, OCR), пользовательским синтезом речи с использованием более 300+ динамических голосов и автоматическим удалением блоков рассуждений (например, `<think>...</think>`) из выходных данных.
*   **Перевод в окне просмотра документов:** Исправлена ошибка, из-за которой перевод не выполнялся для пользователей NVDA с неанглийским интерфейсом; теперь в Google Translate отправляется стандартный двухбуквенный код языка вместо локализованного названия языка.
*   **Повторная попытка пакетного сканирования PDF:** Реализована высокооптимизированная, отдельная и "тихая" логика повторных попыток для пакетного сканирования PDF-документов, чтобы предотвратить повторные загрузки и избежать раздражающих всплывающих окон об ошибках во время повторных попыток.
*   **Состояние в просмотре документов:** Исправлена ошибка, из-за которой общий статус дополнения (проверяемый клавишей `I`) оставался на "Пакетная обработка запущена" во время длительного сканирования документов.
*   **Устранена проблема с потоками:** Исправлено критическое утверждение потока `IsMain() failed in wxTimerImpl`, вызывавшее падение при открытии документов из фонового потока, путём переключения очереди вызовов графического интерфейса на `wx.CallAfter`.

## Изменения в версии 6.1.2

*   **Предварительная проверка дублирования меток:** Исправлена проблема при одиночной разметке, когда проверка дублирования использовала старые координатные ключи, заставляя NVDA делать повторные ИИ-запросы для уже размеченных объектов вместо озвучивания существующей метки.
*   **Чат с документами для не-Gemini провайдеров:** Исправлена строгая проверка API-ключа в чате документа (`on_ask`), чтобы пользователи OpenAI, Groq или локальных пользовательских провайдеров (например, Ollama) могли успешно общаться с документами без блокировки.
*   **Перевод для быстрого Chrome OCR:** Восстановлен бесплатный API перевода для Chrome OCR. Перевод извлечённого текста теперь обходит Gemini AI, экономя квоты API и ускоряя процесс перевода.
*   **Алфавитно-цифровой фильтр CAPTCHA:** Исправлена логика фильтрации в решателе CAPTCHA, чтобы неалфавитно-цифровые символы корректно очищались во всех ситуациях.
*   **Обновление справки уровня команд:** Исправлено сочетание для объявления статуса в меню справки с `L` на `I`, а также добавлены обе команды разметки (`L` и `Shift+L`) в список.

## Изменения в версии 6.1.1

*   **Исправление вывода мышления Gemma 4:** Исправлена проблема с моделями Gemma 4, когда весь внутренний мыслительный процесс отображался как конечный ответ, или когда отключение мышления приводило к пустым ответам. Теперь дополнение правильно изолирует и извлекает только окончательный чистый текстовый ответ.
*   **Пакетный OCR из проводника:** Теперь вы можете выбрать несколько фотографий или PDF-файлов непосредственно в проводнике Windows и извлечь из них текст или проанализировать их в пакетном режиме. Дополнение автоматически отфильтрует и обработает только поддерживаемые форматы файлов.

## Изменения в версии 6.1.0

*   **Универсальная интеграция локального ИИ (Настройка локального ИИ):** Добавлена новая кнопка **"Настроить локальный ИИ"** в настройках пользовательского провайдера. Пользователи теперь могут мгновенно настраивать локальные ИИ-движки, включая **Ollama**, **LM Studio**, **Jan.ai** и **KoboldCPP**.
*   **Интеллектуальный обход локального прокси:** Переработана логика подключения с механизмом обхода прокси. Теперь дополнение достаточно умно, чтобы полностью обходить системные прокси Windows для локальных подключений, обеспечивая стабильные локальные ИИ-подключения даже при активном VPN/TUN-режиме.
*   **Сверхстабильная ИИ-разметка (v2):** Замена абсолютных координат экрана на гибридную систему **"Сигнатура объекта"**. Метки теперь зависят от программных идентификаторов (UIA **AutomationId** или Win32 **ControlID**) и относительных координат окна, что делает ваши пользовательские метки полностью устойчивыми к изменению размера окна, его перемещению, смене монитора или масштабированию.
*   **Бесшовная автоматическая миграция меток:** Обновление происходит полностью незаметно. Дополнение автоматически перенесёт ваши старые метки, основанные на координатах, в новый стабильный формат "отпечатка" в фоновом режиме при первом фокусе, без потери данных.

## Изменения в версии 6.0

    *   *Как это работает:* Теперь вы можете отдавать голосовые инструкции для управления вашим ПК. Например, в полностью недоступном приложении, где ваш скринридер молчит, вы можете нажать **Shift+A** и ввести: *"Нажми на кнопку Настройки"* или *"Найди поле поиска, введите 'Последние новости' и нажмите Enter"*. ИИ визуально определяет элементы, двигает мышь и выполняет задачу за вас.
    *   *Примечание по производительности:* Эта функция оптимизирована для **Gemini 3.0 Flash (Preview)**, обеспечивая невероятно быстрые и умные ответы, способные справляться с самыми сложными интерфейсами.
    *   **⚠️ Предупреждение об использовании API:** Поскольку ИИ-оператор должен "видеть", что именно происходит, чтобы быть точным, он отправляет скриншот высокого разрешения на каждом шаге. ИИ просканирует всё окно и создаст список каждого кликабельного элемента, который он видит, включая значки, графику и меню. Просто выберите элемент из списка, и ИИ-оператор нажмёт на него за вас. Это как иметь "доступный слой" поверх любого приложения.
*   **Контекстно-зависимое умное действие с файлом (F):** Клавиша "F" была полностью переработана. Она больше не предполагает, что вам нужен только OCR. Меню динамически адаптируется в зависимости от типа файла и вашего активного ИИ-движка.
*   **Оптимизация ядра:** Мы провели глубокую очистку внутренней логики дополнения, удалив неиспользуемые устаревшие функции и избыточный код. Это обеспечивает более лёгкий, быстрый и надёжный опыт для всех пользователей.
